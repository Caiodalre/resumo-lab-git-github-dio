# Resumo do Lab - Git e GitHub

Este repositório foi criado como parte do desafio de projeto da DIO, com o objetivo de documentar os principais aprendizados obtidos durante o laboratório sobre Git e GitHub.

## Repositório

Link do projeto no GitHub:

https://github.com/Caiodalre/resumo-lab-git-github-dio

## O que aprendi

Durante o desenvolvimento deste lab, aprendi que o Git é uma ferramenta de controle de versão utilizada para registrar alterações em arquivos ao longo do tempo. Ele permite acompanhar o histórico de modificações, recuperar versões anteriores e trabalhar de forma mais organizada em projetos de desenvolvimento.

Também aprendi que o GitHub é uma plataforma baseada em Git, utilizada para hospedar repositórios, colaborar com outras pessoas, documentar projetos e criar um portfólio profissional.

## Principais conceitos estudados

- Controle de versão
- Repositórios locais e remotos
- Commits
- Branches
- Pull Requests
- Forks
- Issues
- Markdown
- README.md
- GitHub Projects
- GitHub Copilot
- Autenticação de dois fatores
- Chaves SSH

## Comandos Git importantes

git init  
git status  
git add .  
git commit -m "mensagem do commit"  
git branch  
git checkout nome-da-branch  
git merge nome-da-branch  
git clone URL_DO_REPOSITORIO  
git push  
git pull  

## Importância do GitHub

O GitHub é importante porque permite armazenar projetos na nuvem, colaborar com outras pessoas, controlar versões do código e apresentar projetos de forma organizada para recrutadores e empresas.

Além disso, o GitHub contribui para a construção de um portfólio profissional, demonstrando conhecimentos técnicos, organização e evolução nos estudos.

## Conclusão

Este lab foi importante para compreender os fundamentos de Git e GitHub. A partir dele, consegui entender melhor como criar repositórios, documentar projetos, utilizar Markdown e organizar meus estudos de forma mais profissional.
